# NBA Draft API

A robust REST API providing structured NBA player statistics, team data, and contract information.

This project serves as the backend layer for NBA-related applications. It consumes the data generated by the **NBA Data Scraper** and exposes it through a clean, documented, and professional REST interface.

## Key Features

* **Data Upsert Engine:** Automated synchronization from JSON sources that updates existing records and inserts new ones based on unique business keys.

* **Optimized DTOs:** Uses **Java Records**  for memory-efficient, immutable data transfer between layers.
**Live API Documentation:** Integrated **Swagger UI** for instant endpoint testing and schema exploration.

## Technologies Used

* **Java 21** (LTS version)

* **Spring Boot 3.4.5** (Web, Data JPA)

* **MySQL** (Relational storage) 

* **Lombok** (Code simplification)

* **Springdoc OpenAPI** (Interactive Swagger documentation)

## Getting Started

### Prerequisites
* Java Development Kit (JDK) 21.
* Maven installed on your machine.
* A MySQL instance running (locally or remotely).

### Environment Variables
The application is configured to run out-of-the-box with default values, but you can override them using environment variables:

| Variable | Description | Default Value |
|----------|-------------|---------------|
| `DB_URL` | JDBC URL for MySQL | `jdbc:mysql://localhost:3306/nba_draft_db` |
| `DB_USER` | Database username | `root` |
| `DB_PASSWORD` | Database password | `(empty)` |
| `CORS_ORIGINS` | Allowed CORS domains | `http://localhost:4200` |

### Installation

1. Clone the repository:
```bash
git clone https://github.com/julien-croynar/nba-draft-api.git
```

2. Configure your database credentials in **src/main/resources/application.properties.**

3. Run the application:

```bash
./mvnw spring-boot:run
```

## Usage & API Documentation
Once launched, the API is accessible at `http://localhost:8080`.
You can explore the full documentation and test the endpoints via the **Swagger UI:**

 `http://localhost:8080/swagger-ui/index.html`

### Main Endpoints

| Method | Endpoint       | Description                |
|:-------|:---------------| :------------------------- |
| GET    | `/api/players` | Retrieve all players with full details |
| GET    | `/api/players/basic`       | Retrieve a list of players using lightweight DTOs |
| GET    | `/api/teams`       | Retrieve all NBA teams |
| GET    | `/api/players/team/{teamId}`       | Filter players by their specific team code |

## Project Structure
```bash
src/main/java/com/yusuke/nba_draft_api/
├── controller/    # REST Endpoints (Player, Team) 
├── dto/           # Data Transfer Objects (Java Records)
├── model/         # JPA Entities (Player, Team, Stats, Contract)
├── repository/    # Spring Data JPA interfaces
├── service/       # Business logic and Data Initialization
└── NbaDraftApiApplication.java
```

## Data Integration
This API is designed to work hand-in-hand with the NBA Data Scraper (**https://github.com/julien-croynar/nba-data-scraper**)
Simply place your players-data.json and team-data.json in the src/main/resources/ folder to have your database populated automatically at runtime.
